Ecosystem Interoperability

RMath is designed to be a "drop-in accelerator." It implements standard memory protocols to ensure zero-copy compatibility with the Python scientific ecosystem.

Proven: NumPy Bridge

RMath implements the __array__ protocol, allowing standard NumPy conversions without duplicating memory.

numpy_bridge.py 
import rmath.array as ra
import numpy as np

# Create RMath Array
a = ra.ones(2, 2)

# Create Zero-Copy NumPy View
np_view = np.asarray(a)

# Modify through NumPy
np_view[0, 0] = 99.0

# Verify RMath original reflects the change
print(f"RMath a[0,0]: {a.data[0]}")
RMath a[0,0]: 99.0

Proven: PyTorch Interop

Using the DLPack protocol, RMath can pass data to and from PyTorch without costly serialization.

pytorch_interop.py 
import rmath.array as ra
import torch

# RMath -> PyTorch
ra_arr = ra.randn(10)
th_tensor = torch.from_dlpack(ra_arr.__dlpack__())

# PyTorch -> RMath
th_arr = torch.randn(5, 5)
ra_tensor = ra.Tensor.from_dlpack(th_arr)

print(f"PyTorch Shape: {th_tensor.shape}")
print(f"RMath Tensor: {ra_tensor.shape}")
PyTorch Shape: torch.Size([10]) RMath Tensor: [5, 5]
Zero-Copy Philosophy: RMath treats memory as a shared resource. Whether you're using __array_interface__ for legacy libraries or DLPack for modern accelerators, RMath ensures no bytes are wasted.
