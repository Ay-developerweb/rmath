Ecosystem Interop

RMath is a drop-in accelerator, not a replacement. Every Array and Tensor can flow freely between NumPy, PyTorch, JAX, pandas, and scikit-learn — often with zero-copy transfers.

The __array__ Protocol

RMath's Array class implements Python's __array__ protocol. This means any library that expects a NumPy array will automatically convert an rm.Array without you writing a single extra line of code.

protocol.py
import numpy as np
import rmath as rm

a = rm.Array([[1, 2], [3, 4]])

# This "Just Works" — np.array() calls a.__array__() internally
result = np.linalg.eigvals(a)
print(result)  # NumPy receives the data with no manual conversion
How it works: When a library calls np.array(obj) or np.asarray(obj), Python checks if obj has an __array__ method. RMath's implementation returns a properly shaped NumPy array backed by a contiguous memory buffer — typically a single memcpy.

NumPy

MethodDirectionDescription
Array.from_numpy(np_arr)NumPy → RMathImport a NumPy ndarray into an RMath Array.
arr.to_numpy()RMath → NumPyExport to a NumPy ndarray. Shape is preserved.
np.array(arr)RMath → NumPyAutomatic via __array__ protocol.
numpy_bridge.py
import numpy as np
import rmath as rm

# NumPy → RMath
np_arr = np.array([[1.0, 2.0], [3.0, 4.0]])
rm_arr = rm.Array.from_numpy(np_arr)
print(rm_arr)
# Array([
#        [ 1.0000,  2.0000],
#        [ 3.0000,  4.0000]])

# RMath → NumPy (explicit)
back = rm_arr.to_numpy()
print(back)
# array([[1., 2.],
#        [3., 4.]])

# Roundtrip check
assert np.allclose(np_arr, back)  # True — no precision loss

PyTorch

MethodDirectionDescription
Array.from_torch(tensor)PyTorch → RMathImport a PyTorch CPU tensor.
arr.to_torch()RMath → PyTorchExport to a PyTorch tensor. Routes via NumPy for efficiency.
pytorch_bridge.py
import torch
import rmath as rm

# PyTorch → RMath
t = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
rm_arr = rm.Array.from_torch(t)

# Process in Rust (fast)
result = rm_arr.sigmoid().tanh()

# RMath → PyTorch
back = result.to_torch()
print(type(back))  # <class 'torch.Tensor'>

import jax.numpy as jnp

import pandas as pd
import rmath as rm

# Create data in Rust, export to pandas
data = rm.Array([[1, 2], [3, 4], [5, 6]])
df = data.to_dataframe(columns=["x", "y"])
print(df)
#    x    y
# 0  1.0  2.0
# 1  3.0  4.0
# 2  5.0  6.0

# pandas → RMath
back = rm.Array.from_dataframe(df)

Scikit-Learn

Because RMath implements __array__, any scikit-learn estimator that accepts array-like inputs will work with rm.Array directly. No .to_numpy() call needed.

Validation Helpers
MethodReturnsDescription
arr.sklearn_shape()(n_samples, n_features)Returns shape as sklearn convention tuple.
arr.validate_sklearn()NoneRaises if not 2-D, or contains NaN/Inf.
sklearn_bridge.py
import rmath as rm
from sklearn.linear_model import LinearRegression

# Generate data entirely in Rust
X = rm.Array.randn(100, 1)
Y = rm.Array.randn(100, 1)

# Pass directly to sklearn — no conversion needed
model = LinearRegression()
model.fit(X, Y)

print("R² Score:", model.score(X, Y))

# Validate before feeding to stricter estimators
X.validate_sklearn()  # raises ValueError if NaN/Inf/non-2D

Transfer Performance

BridgeMechanismCost
to_numpy()Contiguous buffer → NumPy ndarraySingle memcpy
from_numpy()NumPy ndarray → flat VecSingle memcpy
to_torch()Routes via to_numpy()Single memcpy + torch.from_numpy
to_jax()Routes via to_numpy()Single memcpy + jnp.asarray
__array__Auto-called by any NumPy-compatible libSingle memcpy (implicit)
Why not Python lists? Before v0.1.4, to_torch() and to_jax() converted data to a Python list first — requiring N individual float object allocations. The current implementation routes through NumPy's contiguous buffer, reducing overhead from O(N) Python object allocations to a single memcpy.
