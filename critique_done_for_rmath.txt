## 🛠️ Surgical Improvement Targets (v0.2.0 Roadmap)

While `rmath` leads in spectral and fusion logic, our deep audit identified the following critical areas that require optimization to match or beat MKL/OpenBLAS industrial standards.

### 1. The "View vs Copy" Debt (Memory)
- **Problem**: `Array.transpose()` currently performs a physical memory copy (Measured: +122MB for 4000x4000).
- **Target**: Implement **Strided Lazy Views**. Transposing should be an $O(1)$ metadata change (pointer/stride swap) with **zero memory overhead**.

### 2. The LAPACK Kernel Gap (Linalg)
- **Problem**: Cholesky decomposition is currently **37x slower** than MKL (3.11s vs 0.08s).
- **Target**: Integrate `lapack-src` and use block-based cache-friendly kernels. Avoid naive nested loops in Rust for $O(n^3)$ operations.

### 3. FFI Parallelism (GIL Management)
- **Problem**: Computations on `Vector` do not yet release the Python Global Interpreter Lock (GIL). Multi-threading with standard Python `threading` shows **0.00x speedup**.
- **Target**: Implement `py.allow_threads()` around all compute-heavy Rust loops to enable native parallelization via Rayon.

### 4. SIMD Saturation (Compute)
- **Problem**: `sqrt` is 3.6x faster than NumPy, but fall short of the theoretical **162x** possible with hand-tuned SIMD.
- **Target**: Move from LLVM auto-vectorization to explicit `std::simd` or `sleef-sys` for transcendental functions (`exp`, `sin`, `log`).

### 5. API Consistency & Ergonomics
- **Missing `Vector.to_numpy()`**: Users currently must convert to `Array` or use `tolist()` (slow) to hand back to NumPy.
- **`eigh` Order**: Needs to be aligned with NumPy standard `(evals, evecs)` to simplify migration.

### 6. API Ergonomics & Vectorization Documentation
- **Problem**: Current training scripts (e.g., `train_csv.py`) use non-vectorized Python loops for multi-feature accumulation (`pred = pred + X * W[j]`).
- **Target**: Improve `Tensor` documentation and `__matmul__` support to encourage vectorized `X @ W` patterns, which significantly reduce Python-side overhead and FFI crossings.
